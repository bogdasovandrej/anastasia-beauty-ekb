# Решение проблем с доступом к сайту

## Проблема 1: Локальный сервер не открывается

### Решение:

1. **Откройте браузер и перейдите по адресу:**
   ```
   http://localhost:8000
   ```
   или
   ```
   http://127.0.0.1:8000
   ```

2. **Если не открывается:**
   - Проверьте что сервер запущен (в терминале должно быть "Serving HTTP on :: port 8000")
   - Попробуйте другой порт: `python -m http.server 8001`
   - Проверьте антивирус/фаервол

3. **Проверка в Chrome DevTools (эмуляция Android):**
   - Откройте `http://localhost:8000`
   - Нажмите `F12`
   - Нажмите `Ctrl+Shift+M`
   - Выберите "Samsung Galaxy S20" или "Pixel 5"
   - Проверьте отображение

## Проблема 2: Vercel показывает "Неверная конфигурация"

### Причина:
Домен `anastasia-beauty-ekb.ru` требует правильной DNS-конфигурации.

### Решение по шагам:

#### Шаг 1: Проверьте текущие настройки в Vercel

1. Откройте проект в Vercel Dashboard
2. Перейдите в **Settings** → **Domains**
3. Убедитесь что домен `anastasia-beauty-ekb.ru` добавлен

#### Шаг 2: Настройка DNS на reg.ru

**Вариант А: Использовать Vercel DNS (рекомендуется)**

1. В Vercel: **Settings** → **Domains**
2. Нажмите на домен `anastasia-beauty-ekb.ru`
3. Следуйте инструкциям для настройки DNS
4. Vercel предоставит NS-серверы:
   - `ns1.vercel-dns.com`
   - `ns2.vercel-dns.com`

5. На reg.ru:
   - Войдите в панель управления
   - Найдите домен `anastasia-beauty-ekb.ru`
   - Найдите раздел "DNS-серверы" или "NS-серверы"
   - Измените NS-серверы на:
     ```
     ns1.vercel-dns.com
     ns2.vercel-dns.com
     ```
   - Сохраните изменения

**Вариант Б: Использовать A-записи (если хотите оставить DNS на reg.ru)**

1. В Vercel: **Settings** → **Domains**
2. Нажмите на домен
3. Выберите "Use Vercel's DNS" → Отключите
4. На reg.ru добавьте записи:

   ```
   Тип: A
   Имя: @
   Адрес: 76.76.21.21
   (повторите для 76.76.21.1, 76.76.21.10, 76.76.21.15)
   
   Тип: CNAME
   Имя: www
   Адрес: cname.vercel-dns.com
   ```

#### Шаг 3: Ожидайте распространения DNS

- **Время распространения:** от 1 часа до 48 часов
- **Обычно:** 1-4 часа
- Проверить статус: https://dnschecker.org/

#### Шаг 4: Проверка HTTPS

HTTPS включается **автоматически** после распространения DNS:
- Не нужно включать вручную
- Vercel использует Let's Encrypt
- Обычно занимает 10-30 минут после DNS

## Как проверить что DNS работает

### Способ 1: DNS Checker

1. Перейдите: https://dnschecker.org/
2. Введите: `anastasia-beauty-ekb.ru`
3. Выберите тип: **NS** или **A**
4. Нажмите "Check"
5. Если видите IP Vercel (76.76.x.x) — DNS работает

### Способ 2: Командная строка

```cmd
nslookup anastasia-beauty-ekb.ru
```

Или:

```cmd
ping anastasia-beauty-ekb.ru
```

### Способ 3: Vercel Dashboard

1. Откройте Vercel Dashboard
2. Перейдите в **Projects** → ваш проект
3. Перейдите в **Settings** → **Domains**
4. Статус должен быть "Active" (зеленый)

## Временное решение: использовать Vercel домен

Пока настраиваете основной домен, используйте временный:

1. В Vercel Dashboard откройте проект
2. Вверху увидите домен вида: `your-project.vercel.app`
3. Этот домен работает сразу после деплоя
4. Можно использовать для тестирования

## Чеклист для Vercel

Перед добавлением домена убедитесь:

- [ ] Проект деплоится без ошибок

## Частые ошибки

### Ошибка: "This site can't be reached"
**Причина:** DNS еще не распространился
**Решение:** Подождать 1-4 часа

### Ошибка: "Too many redirects"
**Причина:** Конфликт HTTP/HTTPS
**Решение:** Очистить кэш браузера, подождать HTTPS

### Ошибка: "404 Not Found"
**Причина:** Неправильная структура файлов
**Решение:** Убедитесь что `index.html` в корне проекта

### Ошибка: "520 Origin Error"
**Причина:** Проблемы с origin сервером
**Решение:** Проверить деплой в Vercel, пересоздать деплой
